## 2. Scan + search hardening ("must work at all cost")

### 2.1 Scan reliability checklist
- Wrap every TMDb call in `withRetry(3, exponentialBackoff)`.
- Treat zero-byte and `*.part` / `*.crdownload` files as **skip**, not error.
- On any per-file failure: log to `action_history` with `FileAction='scan_error'`, **continue** the loop, increment `failed_count`. Final exit code stays 0 if `succeeded_count > 0`.
- Add `--strict` flag for CI: any failure → non-zero exit.

### 2.2 Search reliability checklist
- New chain order in `tmdb/fallback.go`:
  1. `SearchMulti(title + year)`
  2. `tryProgressiveTrim`
  3. **NEW** `tryOmdb` (if `OMDB_API_KEY` set) — see §3
  4. **NEW** `tryImdbScrape` (no key, last resort) — see §3
  5. existing `tryImdbViaWeb` (DuckDuckGo + cache)
- Every step times out at 10 s; chain total budget 25 s.
- All providers cache into the same `imdb_cache` table.

## 3. Multi-source fallback — OMDb + IMDb scrape

### 3.1 OMDb provider (`tmdb/omdb.go` — new)
```go
type OmdbClient struct { APIKey string; HTTP *http.Client }
func (o *OmdbClient) SearchByTitle(title string, year int) ([]SearchResult, error)
func (o *OmdbClient) GetByImdbId(imdbID string) (*SearchResult, error)
```
- Endpoint: `https://www.omdbapi.com/?apikey=KEY&t=TITLE&y=YEAR`
- Map OMDb fields → existing `SearchResult{ID, Title, Year, Overview, MediaType, ImdbID}`. `ID` (TMDb numeric) stays 0 — caller uses `LookupByImdbId` to enrich.
- Config key: `OmdbApiKey` in `config` table; env var `OMDB_API_KEY` as fallback.

### 3.2 IMDb HTML scrape (`tmdb/imdb_scrape.go` — new)
- Endpoint: `https://www.imdb.com/find/?q=TITLE+YEAR&s=tt&ttype=ft`
- Parse with `regexp` (no HTML deps): extract first `tt\d{7,10}` from result list.
- User-Agent: `Mozilla/5.0 (compatible; movie-cli/2.x; +https://github.com/alimtvnetwork/movie-cli-v7)`.
- 10 s timeout. On any error → return nil, log warning, do NOT bubble error.
- Result feeds into existing `lookupByImdbId` for TMDb enrichment.

### 3.3 Wiring
- `tmdb/fallback.go` `SearchWithFallback` becomes a small dispatcher reading a `[]Provider` slice. Each provider implements `Try(title, year string) []SearchResult`.
- This replaces hard-coded ordering and makes adding Trakt/etc. trivial later.

## 6. Risks & mitigations

| Risk | Mitigation |
|------|------------|
| OMDb daily limit (1k req/day free tier) | Cache hits in `imdb_cache`; provider only invoked on TMDb miss |
| IMDb HTML structure change breaks scraper | Regex matches `tt\d{7,10}` only — robust to layout changes; falls through silently on no match |
| E2E hits real TMDb network in CI | All TMDb calls in `e2e_test.go` use `httptest.NewServer`; smoke script also points at fake via `TMDB_BASE_URL` env |
| Shell smoke fails on Windows runner | Provide `scripts/e2e-smoke.ps1` mirror; CI matrix runs only on `ubuntu-latest` initially |
| `--all` move catastrophe (wrong dest) | Mandatory `[y/N]` prompt showing count + first 3 paths + `--yes` only opt-in |
| Sidecar writes fail mid-scan | Atomic `tmp` + `os.Rename`; failure logged but does **not** abort the scan |
| CI commits back & loops | Workflow has zero `git push` / `git commit` steps; ref `mem://issues/04-ci-log-commit-loop` |
